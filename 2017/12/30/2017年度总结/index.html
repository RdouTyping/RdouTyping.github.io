2017年度总结

发表于 2017-12-30 | 分类于 lifestyle

这么快又一年结束了，今年收获可以说是满满：生活上收获了一个可爱的小宝宝，买了房；工作上我升了一级，老婆的H1B也搞定了；同时，经过一年的投资，赚了一点钱，但更多的是满满领悟到了该如何进行投资。

生活方面

今年可以说是我生活方面进步最快的一年，2月份老婆怀孕，11月第一个儿子“小豆包”出生，一切都是那么的顺利。在老婆怀孕的中旬，我们又在8月初买到了人生第一套房子。从看房到买房大概经历了3个月，这段时间见证了老婆肚子由小变大，其中的喜悦不言而喻。虽然说“小豆包”出生后生活变的很忙碌，但看着“小豆包”一天天的长大，看着他可爱的样子，心情就会好起来，忙碌什么的都不是事儿了。除了“小豆包”和买房，今年还有两件事情值得庆祝，第一就是老婆从ITU毕业了，顺利拿到software engineering的学位；第二就是老婆今年的H1B终于抽中并批准了。恭喜老婆了，我们这一年多没白跑北加，H1B拿下也意味着往北加跑的日子结束了，一切又回归了平静。如果要我给2017年打分，生活方面我可以给满分！

工作方面

这一年工作上成就并不多，大部分工作还是按部就班。但今年公司的变化让我的工作也有了一些不同，首先组里走了两名员工。2017年开始的几个月，因为组里基本没有项目，因此在那几个月时间我利用上班时间开发了一套简单的FTL，这大概花了我3个月时间，包括看paper、写代码以及debug。在我完成beta版本后，本打算继续完成下一个版本的FTL，哪知道组里发生了人员变动。

第一个员工Y走后，她之前做的SATA相关的模块本来老板是打算让我做的，我花了一个月时间学习了SATA协议以及相关的代码、文档，但之后并没有人让我做这个，因此我感觉这段时间是被浪费了。之后组里负责FCT的员工K换到其他组了，他工作的变动把我推到了组里的第一线，这其实对我来说是一件大好事。因为FCT是组里的核心模块，我之前一直想参与进来，但我和之前负责这个模块的同事并没有很好的合作，因此这几年并没有学到多少东西。现在他走了，我正好利用这段时间提高自己。也许老板并没有对我抱有太大的希望，并且由于新的project要明年才能有个雏形，因此给了我充足的时间来做这个。同时，老板给我介绍了一个做FCT的大牛S每天进行培训指导，S人很热情，有问题也基本能回答，跟着他学到了不少东西。但我同时也明白，不可能什么都靠别人，自己搞懂了学会了才行；并且这个时候老板希望我们能在simulation环境下实现测试，但我们根本不可能也没能力将S的测试系统直接放到simulation环境下运行，因此，我自己开发了一个简单的测试程序，通过开发这个测试程序，我自己也对FCT的细节有了更深刻的理解。

如果只给上面这些打分，我可以给9分，似乎一切都向着好的方向发展。但剩下的部分我就不能打这么高的分了。我之前并没有说为什么第一名员工会跳槽，除了私人原因外，还有一个很重要的原因：我的公司不能很快的办理绿卡！为什么？因为公司这一两年来不断的裁员，只要有裁员我的绿卡申请就会拖延半年，断断续续的裁员导致的结果就是入职3年多了，绿卡办理还停留在第一步。因此，对于这部分，我只能给0分。如果绿卡不能办成，之后会很麻烦，因此我在2018年必须做出艰难的选择：离开现在的舒适圈，去一个能办绿卡的陌生环境。同时，因为“小豆包”的出生外加房贷的压力，现在每个月的工资有点不够用了，所以2018年一定要做一些改变：

离开现在的舒适圈，去一个更广阔的天地！

个人提升

今年我学会了什么新东西？

  1. SSD和FTL
  2. 操作系统
  3. SATA
  4. C语言
  5. 投资理财
  6. 读书

第一项是在工作中学习的，最开始我并不了解NAND、SSD以及FTL，但通过阅读paper和查阅网上的文档以及之前同事的代码，对这部分有了深入的理解。当然还不能算上是专家，但其中的基本知识还是懂的。如果要我马上工作在一个新的项目上，我应该也可以立刻上手写代码了。

除了工作中学到最重要的SSD和FTL，这一年我花最多时间学习的就是操作系统了。这其实是我第二次学操作系统了，我是在学堂在线上面听清华的操作系统课程，这门课真的很棒，我也学到了很多东西，可惜后面时间不够，这次又没有完成学业，不过这次基本完成了80%了，比上次好多了。等我跳槽后，我希望能把这门课搞定！学了操作系统后对各种地址映射、中断、线程的切换等有了更深入的了解。之前，完全想不明白中断和线程的切换具体怎么实现的(当然，实验中的实现方法也不是最好的)。总之，除了不能直接在面试时使用外(当然有些职位要求Linux内核开发可能可以算)，学这门课绝对值了！

SATA，学了大概一个月，现在没太多印象了，但基本的数据传输、纠错等还是有了一定的了解，如果以后需要使用应该可以比较快的上手。比较遗憾的地方是没有接触过任何SATA的代码，不管怎样，以后有缘再用吧。

今年花了不少时间读了几本经典的C语言书籍《The C Programming Language》、《c traps and pitfalls》、《Pointers on C》以及经典的软硬结合的书籍《Computer Systems: A Programmer’s Perspective Beta Version》并且上了华盛顿大学的”CSE 351: The Hardware/Software Interface“(使用的教材就是前面这本书)。这些书和课程都大大的提高了我对C语言的理解，同时为了帮助别人也为了提高自己，我在segmentfault这个网站上不停的回答C语言相关的问题。本来要坚持发到年底的，结果到7月8月的时候就完全停下来了，那段时间实在是太忙了，后来闲了下来又不想发了，今年就这样吧，明年再接再厉！

今年我在投资理财方面进步很大，想想之前第一次炒美股时，亏的稀里糊涂，现在想明白了，赚慢钱，作时间的朋友。今年收益不高，但投资的基本理念逐渐成熟了。不炒IPO、不做医药股、不盲目加杠杆、不买垃圾公司、买指数基金等等都是今年也是之后我的投资理念。明年开始，我会坚持发投资理财方面的文章，包括公司分析、国外文章翻译、我的组合等等等等，作时间的朋友，希望自己能在这条路上越走越远(2017年错过了比特币，但我学到了作时间的朋友，用时间来证明我是对的吧)。

今年我到底读了多少书呢？说真的，不记得了，大概20来本吧，比较杂。既有历史相关的比如《龙床》、《大明王朝的七张面孔》，也有经济学相关的《斯坦福极简经济学》、《中国人为什么勤劳而不富有》，还有理财相关的《穷爸爸富爸爸》、《人人都看得懂的财报书》、《股票作手回忆录》，还有杂书比如《稀缺》、《逆转》、《黑天鹅》、《联盟》、《万万没想到：用理工科思维理解世界》、《作时间的朋友》等等。有啥收获？肯定有，但又不能完全说出来，也许这些书都不是直接拿来用的书，因此并不会马上给我带来好处。看书嘛，不能完全看有没有用，因为很多东西不是直接的。明年我还会继续看我喜欢的书，但我应该不会参加什么读书营等，也不会去学一些读书的方法，嗯，我的的慢，我不着急。

哪些我打算学但学或者没坚持下来：

  1. 学英语
  2. 锻炼身体
  3. 坚持写日志
  4. 学RTOS
  5. 学PCIe

其实前3项我基本都坚持了3个月，但之后就慢慢放弃了，除了惰性其实还有些其他原因：比如锻炼我受伤了两次，导致直接停了1个月；英语，嗯，不知道为什么就不想听了；写日志有时候没时间然后就忘了。不管怎样，这些都没坚持超过3个月，我希望我明年能多坚持一些时间，这算是一个小小的计划吧。后两项没做也是有原因的，RTOS是一直没时间去学，网上买了课程(好像买了)但没时间去学；而PCIe就是因为用不到外加没时间，所以这两项就搁置在一边了，明年可能要学RTOS但PCIe可能还是不会去碰。

总体而言，这部分我打7分吧，还有很多可以提高的地方，明年继续加油！

03/2017

  1. 3月24号第一次去医院看到了“小豆包”，特别小，特别可爱。当时我们还觉得生娃还早，现在看来时间过得真快，“小豆包”现在都已经40多天了。

05/2017
